# priority_algorithm.md — 우선순위 산정 + 시간 배분

**원칙**: 면접 단계(1차/2차) + 시간 제약 + 매칭 결과를 종합하여 **시간 내 가장 중요한 질문 N개**를 선별. 학술 근거: *Cohen 1988* — power analysis 기반 표본 크기 산정 원리 차용.

---

## 0. 결정적 시간 배분 알고리즘 (v0.1.4, Critical)

같은 직무·시간 입력은 같은 시간 분포를 만들어야 한다 (재현성). 운영 데이터 분석 결과 LLM의 자유 시간 재할당이 후보자 비교를 깨뜨렸으므로 다음 결정적 공식을 강제한다.

### 0.1 공식

```
입력: minutes (1차 또는 2차 면접 시간)

오프닝 = 5분
클로징·후보자 질의 = 5분
MVC 블록 = 5가치 × per_value_minutes (per_value_minutes는 stage·minutes에 따라 결정, 0.2 참조)
직무역량 블록 = minutes - 10 - MVC 블록

직무역량 시간 분배 (가중치 비례):
  C1_minutes = round(직무역량 블록 × C1_weight)
  C2_minutes = round(직무역량 블록 × C2_weight)
  ...
  단 anchor 질문이 있는 역량은 최소 5분 보장 (시간 부족 시 비-anchor 역량부터 단축)
```

### 0.2 MVC 가치당 시간 (per_value_minutes)

| stage | minutes | per_value_minutes | MVC 총 |
|---|---:|---:|---:|
| 1차 | 60 | 3분 | 15분 |
| 1차 | 75 | 3분 | 15분 |
| 1차 | 90 | 4분 | 20분 |
| 1차 | 120 | 5분 | 25분 |
| 2차 | 60 | 6분 | 30분 |
| 2차 | 90 | 8분 | 40분 |
| 2차 | 120 | 10분 | 50분 |

위 표에 없는 시간은 가장 가까운 값으로 보간한다. 단 **MVC 5가치 전부 평가는 절대 금지 해제 안 됨**.

### 0.3 시간 부족 처리

직무역량 블록이 음수 또는 6분 미만이면 다음 순서로 사용자에게 경고한다.

1. "면접 시간이 부족합니다. MVC 5가치 강제 평가를 위해 최소 30분(1차) 또는 50분(2차)이 필요합니다."
2. 시간 증가 또는 stage 변경 권고.
3. 사용자 강행 시 클로징을 3분으로 축소하고 직무역량 분배를 진행한다.

### 0.4 비-결정성 차단 가드

LLM이 다음 자유를 행사하면 즉시 거부한다.

- 가치 선별 (MVC 5가치 중 일부만 평가) → 거부
- 추가 자유 영역 (직무 역량 N개 + 5가치 외) → 거부
- 검증 우선순위 라벨 (High/Medium/Critical) → 거부 (매칭 강도로 일원화)
- 시간 자유 재할당 → 거부 (위 공식만 사용)

## 0.5. Anchor 질문 정책 (v0.1.3 추가, v0.1.4 강화)

각 직무에 **모든 후보자가 공통으로 답하는 anchor 질문 1~2개**를 강제 포함한다. 이는 *Levashina et al. (2014) 동일 질문 원칙*을 강화하여 후보자 간 직접 비교 가능성을 확보한다.

### 0.5.1 Anchor 선정 규칙

직무별 anchor 질문은 다음 규칙으로 자동 결정한다.

| 우선순위 | 규칙 |
|---:|---|
| 1 | 가중치 **1위 역량의 BEI 1번 질문** — 무조건 anchor |
| 2 | 가중치 **2위 역량의 BEI 1번 질문** — 1위와 가중치 차이가 5% 이내일 때 anchor 추가 (최대 2개) |
| 3 | MVC 모듈은 anchor 별도 지정 불필요 — 5가치 모두 매번 공통으로 평가됨 |

### 0.5.2 Anchor 질문의 처리 원칙

CV 매칭 강도와 **무관하게** 우선순위 산정에 자동 포함한다.

매칭이 weak 또는 none인 경우에도 anchor BEI는 제외하지 않는다. 단 검증 의도가 자동으로 다음과 같이 전환된다.

| 매칭 강도 | Anchor의 검증 의도 |
|---|---|
| 🟢 strong | 강점 진위·정량 검증 |
| 🟡 moderate | 확장 검증 |
| 🟠 weak | 학습 의지·잠재력 평가 |
| 🔴 none | 잠재력·문제 해결 접근법 평가 |

### 0.5.3 출력 표시

각 anchor 질문에는 출력에 `🔗 공통 Anchor` 배지를 부착한다. 매칭 매트릭스·진행 계획 표의 "Anchor" 컬럼에 🔗 표시한다.

### 0.5.4 환각 가드

priority_algorithm 실행 시 anchor 질문이 결과에 누락되면 즉시 재산정한다. 우선순위가 시간 제약으로 인해 anchor를 제외하려는 경우, **비-anchor 질문을 먼저 제거**한다.

## 0.6. 필수 / 선택 질문 구성 (v0.2.0 추가, Critical)

면접관이 빠르게 운용하고, 동시에 후보자 간·세션 간 구조화 일관성이 끊기지 않도록, 직무역량 질문을 **필수(must-ask)**와 **선택(optional)**으로 명확히 이원화한다.

### 0.6.1 구성 규칙 (결정적)

| 구분 | 개수 | 선정 규칙 | 의미 |
|---|:---:|---|---|
| 🔴 **필수** | **정확히 2개** | §0.5의 anchor 2개 (가중치 1·2위 BEI). anchor가 1개뿐이면 가중치 2위 역량 BEI를 필수로 승격 | 모든 후보자·모든 면접 세션 공통 — 비교 가능성·연속성 보장 |
| ⚪ **선택** | **2~3개** | 필수 제외 후 priority_score 상위. stage·시간에 따라 60분=2개, 90분 이상=3개 | 면접관이 시간·후보 상황에 맞게 취사 |

필수 2개는 **모든 후보자가 반드시 받는다**. 선택은 면접관 재량이되, 누적 가중치 80% 커버를 우선 충족하도록 정렬한다 (§3.3).

### 0.6.2 출력 표시

직무역량 질문 섹션을 `🔴 필수 질문 (2)` / `⚪ 선택 질문 (2~3)` 두 하위 묶음으로 분리해 렌더한다. 각 질문 카드 제목에 `🔴 필수` 또는 `⚪ 선택` 배지를 부착한다. MVC 5가치는 전부 필수(선별 금지)이므로 별도 배지 없이 5개 모두 렌더한다.

### 0.6.3 환각·결정성 가드

- 필수 2개를 임의로 늘리거나 줄이지 않는다 (정확히 2개).
- 시간 부족으로 질문을 줄일 때 **선택 질문부터** 제거하고 필수 2개는 보존한다.
- 필수/선택 분류는 가중치·anchor 규칙으로 결정되며 LLM이 임의 지정하지 않는다.

## 2. 단계별 시간 배분 — §0 단일 기준 (구 버전 표 폐기)

시간 배분은 **§0 결정적 공식만** 사용한다. 과거 버전의 비율 기반 표(1차 직무 70%/MVC 20%, 2차 MVC 50% 등)는 §0과 충돌하여 **폐기되었다**. 어떤 문서·출력에서도 비율 기반 자유 배분을 사용하지 않는다 (§0.4 비-결정성 차단 가드).

- 오프닝 5분 + 클로징 5분 고정 (§0.1)
- MVC = 5가치 × per_value_minutes (stage·minutes 기준 §0.2 표)
- 직무역량 = 잔여 시간, 가중치 비례 분배 (§0.1)

## 3. 직무역량 우선순위 알고리즘

### 3.1 우선순위 점수

각 직무 질문에 대해:

```
priority_score = weight × strategy_multiplier × stage_factor

where:
  weight = competency.weight (0~1)
  strategy_multiplier:
    strong (검증) = 1.0      ← 강점은 진위 확인 가치 큼
    moderate (확장) = 0.9
    weak (갭 탐색) = 0.7
    none (잠재력) = 0.5
  stage_factor:
    1차 직무역량 강점·핵심: 1.0
    1차 갭 탐색: 0.7
    2차 직무 보강: 미해결 영역만 → 1.0 / 그 외 0.3
```

### 3.2 BEI vs SIT 선택 규칙

| 매칭 강도 | 권장 |
|---|---|
| 🟢 strong | BEI 우선 (구체 사례 검증) |
| 🟡 moderate | BEI 우선 + 시간 허락 시 SIT |
| 🟠 weak | SIT 우선 (학습 의지·잠재력) |
| 🔴 none | SIT 우선 + 학습 능력 BEI |

### 3.3 핵심 규칙 — 가중치 합계 80% 커버

상위 priority_score 질문을 골라 **누적 가중치 80% 이상**이 되도록 선택. 그 이하 가중치 역량은 시간 허용 시 추가.

## 4. MVC 우선순위

### 4.1 1차 면접 (크로스체크)

가치당 1문항 = 5문항. **가치별 1차 권장 시드**:

| 가치 | 1차 권장 |
|---|---|
| VALUE_1 | Q-1 (BEI) |
| VALUE_2 | Q-2 (BEI) |
| VALUE_3 | Q-3 (BEI) |
| VALUE_4 | Q-4 (BEI) |
| VALUE_5 | Q-5 (BEI) |

(가치명은 조직 핵심가치로 교체 — `mvc/_core_values.md` / `mvc_values` 테이블) → 직무 친화 컨텍스트 자동 부착 (`mvc_position_context` 또는 _core_values §4 매트릭스 fetch)

### 4.2 2차 면접 (심층)

가치당 BEI 2 + SIT 1 = 3문항 풀에서 **2~3개 선택**. 후보자 1차 응답 데이터(있으면) 또는 매칭 결과로:

- 1차에서 약점 보였던 가치 → SIT 우선 (재검증)
- 1차에서 강점 보였던 가치 → BEI 다른 각도
- Red Flag 후보 가치 → SIT으로 시나리오 검증

## 5. 시간별 권장 질문 수

### 5.1 분당 질문 환산

- 직무 BEI: 본 질문 + 꼬리질문 4~6 = **약 8~12분**
- 직무 SIT: 본 질문 + 꼬리질문 3 = **약 5~8분**
- MVC: 본 질문 + 꼬리질문 2~3 = **약 4~6분**

### 5.2 단계×시간별 권장 (1차 면접)

| 시간 | 직무 BEI | 직무 SIT | MVC | 합계 |
|---:|---:|---:|---:|---:|
| 60분 | 3 | 1 | 3 | **7** |
| 90분 | 4 | 2 | 5 | **11** |
| 120분 | 5 | 3 | 5 | **13** |

### 5.3 2차 면접

| 시간 | MVC 본 | MVC SIT | 직무 보강 | 리더십 | 합계 |
|---:|---:|---:|---:|---:|---:|
| 60분 | 5 | 2 | 1 | 1 | **9** |
| 90분 | 5 | 5 | 2 | 2 | **14** |
| 120분 | 5 | 5 | 4 | 3 | **17** |

## 6. 산출 — 우선순위 정렬된 질문 리스트

```json
{
  "stage": "1차",
  "total_minutes": 90,
  "questions": [
    {
      "order": 1,
      "block": "opening",
      "minutes": 5,
      "content": "오프닝·라포 형성"
    },
    {
      "order": 2,
      "block": "job_competency",
      "minutes": 12,
      "competency_key": "C1_...",
      "question_id": 123,
      "question_type": "BEI",
      "priority_score": 0.250,
      "match_strength": "strong",
      "rationale": "가중치 25% + 강점 검증 우선"
    },
    ...
    {
      "order": N,
      "block": "mvc",
      "minutes": 5,
      "competency_key": "VALUE_1",
      "question_id": 1,
      ...
    },
    {
      "order": N+1,
      "block": "closing",
      "minutes": 5,
      "content": "후보자 질문·다음 단계 안내"
    }
  ],
  "total_question_minutes": 80,
  "buffer_minutes": 10
}
```

**buffer_minutes**: 답변이 길어질 때를 위한 여유 (10~15% 권장).

## 7. 환각 가드

- 매칭 결과 없는 질문은 우선순위 산정에 포함 안 함
- "이 후보자에게 이 질문이 좋을 것"은 추측 — 매칭 강도 + 가중치만으로 산정
- 후보자 성향 추정 금지 (CV에 명시되지 않은 한)
