# cv_parsing.md — CV 파싱 + 개인정보 가드

**원칙**: CV는 민감 개인정보. **세션 내에서만**, 출력 .md에는 발췌만, 메모리·DB·로그 영속 저장 금지.

---

## 1. 입력 형식별 파싱

| 형식 | 처리 |
|---|---|
| **텍스트 붙여넣기** | 그대로 사용. 줄 단위로 인덱싱(`L1`, `L2`...) 인용 추적용 |
| **`.pdf`** | `pdftotext`(poppler) 또는 PyPDF2 → 텍스트. 레이아웃 손실 시 사용자 확인 |
| **`.docx`** | python-docx로 paragraphs + tables 추출 |
| **`.txt` / `.md`** | 그대로 |
| **이미지(.png/.jpg)** | 거부 — OCR은 환각 위험 (사용자에게 텍스트 변환 후 재요청) |
| **링크(Notion 등)** | 본 v0.1 미지원. 사용자에게 텍스트 추출 요청 |

### CV 길이 검증
- < 200자 → "CV 내용이 충분하지 않습니다. 보강 부탁드립니다"
- > 30,000자 → "이 CV는 매우 깁니다. 핵심 부분(최근 5년)만 추출할까요?" 확인

## 2. 추출 구조 (스키마)

```json

  "experience_years": "number",
  "current_role": {"title": "...", "company": "...", "duration": "..."},
  "previous_roles": [
    {"title": "...", "company": "...", "duration": "...", "responsibilities": ["..."]}
  ],
  "skills": {
    "languages": ["..."],
    "frameworks": ["..."],
    "tools": ["..."],
    "domains": ["..."]
  },
  "projects": [
    {
      "name": "...",
      "duration": "...",
      "role": "...",
      "technologies": ["..."],
      "description": "...",
      "outcome": "...",        // 정량 성과
      "cv_lines": ["L42", "L43", "L44"]  // 인용 추적
    }
  ],
  "education": [
    {"degree": "...", "school": "...", "duration": "..."}

  "achievements": [
    {"title": "...", "year": "...", "evidence": "...", "cv_lines": ["L78"]}
  ],
  "publications": ["..."],     // 논문·블로그·컨퍼런스
  "certifications": ["..."]
}
```

## 3. 개인정보 가드 (Critical)

### 3.1 추출하되 출력 .md에 포함 안 함

다음은 **메모리에서만 보유, 출력 .md에 포함 금지**:

- 주소 (전체 또는 시·구 단위)
- 주민등록번호
- 전화번호
- 이메일 (전부)
- 가족 관계
- 사진
- 취미·종교·정치 성향 (직무와 무관)
- 출신 지역·고향
- 결혼 여부

### 3.2 추출 자체 금지

- 주민등록번호 뒷자리 (있으면 즉시 마스킹)
- 신용카드·은행 계좌 정보
- 의료 정보·장애 정보 (직무 직접 관련 시 사용자가 별도 처리)

### 3.3 출력 .md 포함 허용

- 후보자 식별 이름 (또는 약어)
- 경력 연차
- 직무·기술·프로젝트
- 정량 성과
- 학위 (학교 이름은 유사성 편향 위험 — 옵션)

## 4. 인용 추적 (Source Tracking)

CV의 모든 추출 정보에 `cv_lines` 또는 `cv_excerpt` 메타 부착:

```json
{
  "fact": "React 5년 경험",
  "cv_lines": ["L23", "L24"],
  "cv_excerpt": "2020-2025 ... React + TypeScript 기반 ..."
}
```

→ 매칭·인용 시 **반드시 이 메타가 있어야 함**. 없으면 환각 의심 → 즉시 폐기.

---

## 5. 환각 방지 가드

### 5.1 CV에 없는 정보 추측 금지

- ❌ "이 후보자는 React 5년 = 보통 컴포넌트 설계 능숙"
- ✅ "CV에 ''컴포넌트 라이브러리 직접 설계'' 명시" (CV 인용)

### 5.2 일반화 금지

- ❌ "삼성 출신 = 보통 코드 리뷰 문화 익숙"
- ✅ "본 CV에 코드 리뷰 경험은 명시되지 않음"

### 5.3 모호 표현 검출

CV에 다음 표현이 있으면 "확인 필요" 마킹:
- "다양한", "여러", "주도적으로", "협업하여" — 정량 정보 없으면 면접에서 검증
- "성공적으로 마무리" — 결과 정량 없으면 검증
- "산업 표준" 자칭 — 외부 검증 없으면 의심

---

## 6. 다국어 처리

- 한국어 CV → 그대로
- 영어 CV → 그대로 + 한국어 면접 가이드 출력 (사용자 요청 lang)
- 혼용 CV → 두 언어 모두 추출, 출력은 사용자 요청 lang

---

## 7. 세션 폐기 절차

스킬 종료(또는 다음 후보자 시작) 시:

```python
# 의사 코드
del cv_text          # 메모리 폐기
del cv_structured
del cv_temp_file     # 업로드 파일 삭제
gc.collect()         # 즉시 회수
```

사용자에게 알림: "이 후보자의 CV 데이터는 세션 종료 시 폐기되었습니다. 평가 기록은 출력 .md에서만 확인 가능합니다."
