# bars_anchoring.md — BARS 1~5점 행동 앵커 작성 규칙

**원칙**: BARS(*Behaviorally Anchored Rating Scale*, Smith & Kendall 1963)는 각 점수에 **구체적 행동**을 앵커링하여 면접관 간 평가 일관성(Inter-rater Reliability)을 극대화하는 도구. 본 문서는 직무 중립 알고리즘.

---

## 1. BARS 5점 척도 의미

| 점수 | 의미 | 분포 (정규분포 가정) |
|---|---|---|
| **5** (탁월) | 기대를 명확히 초과, 산업·조직 표준 수립 수준 | 상위 5~10% |
| **4** (강함) | 역할 기대를 충족하며 우수 사례 다수 | 상위 25% |
| **3** (충족) | 역할의 기본 기대 충족 | 중간 50% (모드) |
| **2** (약함) | 역할 기대 미달, 발전 필요 | 하위 25% |
| **1** (매우 약함) | 역할 기대 크게 미달, 개선 시급 | 하위 5~10% |

**컷오프**: 면접 의사결정에서 평균 점수 ≥ 3.0은 채용 가능선. 단, 핵심 가치(MVC)는 별도 Go/No-Go.

---

## 2. 앵커 작성 알고리즘

각 역량마다 5개 앵커를 다음 순서로 작성:

### Step 1: 역할 정의 (3점부터 시작)

3점 앵커는 **JD의 ''필수''에 명시된 기본 기대**를 그대로 표현.

예시 패턴:
- "{기본 기능}을 독립적으로 수행 가능"
- "{역할}의 표준 시나리오에서 합리적 의사결정"

### Step 2: 4점 — 3점 + α

3점 충족에 더해:
- 트레이드오프 인지·관리
- 팀 표준·가이드라인 수립 참여
- 정량 지표 기반 개선

### Step 3: 5점 — 산업·조직 표준 수립 수준

- 외부 발표·표준 제안 가능
- 새로운 패러다임·방법론 창출
- 후속 인력 양성·멘토링

### Step 4: 2점 — 3점에서 일부 결손

- 가이드 받으면 가능
- 일부 영역 미흡
- 의사결정 시 시니어 의존

### Step 5: 1점 — 기본 기대 크게 미달

- 핵심 개념 부재
- 기본 도구·기법 미숙
- 책임 회피

---

## 3. 앵커 품질 기준

| 기준 | 설명 |
|---|---|
| **행동 묘사** | 추상 형용사 대신 구체 행동 ("우수함" ❌ → "팀 표준 수립" ✅) |
| **차등 명확** | 5점과 4점, 4점과 3점이 한눈에 다름. 미묘한 차이 금지 |
| **단일 차원** | 한 점수 앵커에 여러 능력 병기 금지 ("X가 능숙하고 Y도 잘하며..." ❌) |
| **JD 정합** | 모든 앵커가 JD의 책임·자격에 매핑 가능 |
| **간결** | 한 앵커 = 1~2 구. 길어지면 평가 시 오용 |

---

## 4. 차등 강도 검증 (Anti-collapse)

다음 패턴 발견 시 재작성:

- 4점과 3점 차이가 모호: "능숙하다" vs "잘한다" 같은 동의어
- 5점과 4점 모두 "전문가 수준" 표현: 차별화 부족
- 1점과 2점 모두 부정 표현 ("부족", "미흡")만 차이: 구체 행동 명시
- 모든 앵커가 같은 길이: 차등 의식 부족 가능성

## 5. JSONB 저장 형식

```json
{
  "5": "산업·조직 표준 수립, 외부 발표 수준",
  "4": "독립 수행 + 팀 표준 수립 참여",
  "3": "기본 역할 독립 수행",
  "2": "가이드 받으면 가능, 일부 영역 미흡",
  "1": "기본 개념 부재, 시니어 의존"
}
```

`interview_questions.bars_anchors` 컬럼에 직접 저장.

## 6. 직무 중립 시드 패턴

다음 5점 앵커 시드는 직무 무관:

| 점수 | 일반 시드 | 변환 시 |
|---|---|---|
| 5 | "{역량 영역}의 산업·조직 표준 수립, 외부 발표·후속 인력 양성 수준" | {역량} 자리에 JD 추출 역량명 |
| 4 | "{역량}을 독립 수행 + 팀 표준 수립·정량 개선" | |
| 3 | "{역량}의 기본 기대를 독립 수행 가능" | |
| 2 | "{역량} 가이드 받으면 가능, 일부 영역 미흡" | |
| 1 | "{역량}의 기본 개념·기법 부재, 시니어 의존" | |

→ 시드를 그대로 쓰지 말고, JD의 구체 어휘로 augment. 단, 기본 5단계 차등은 보존.

## 7. 레벨별 BARS 차등 (v0.2.0 정식)

같은 점수라도 레벨(junior/mid/senior)별로 기대 행동이 다르다. 예: senior의 3점(충족)은 junior의 4점(강함)보다 높은 절대 기대치를 가질 수 있다. *Dreyfus & Dreyfus (1980)* Novice→Expert 5단계를 차등 기준으로 삼는다.

### 7.1 레벨 정의 (Dreyfus 기반)

| 레벨 | 정의 | JD 신호 |
|---|---|---|
| **junior** | Novice~Advanced Beginner. 규칙·가이드 하에 수행 | "0~2년", "주니어", "신입/주니어 환영" |
| **mid** | Competent~Proficient. 독립 수행 + 트레이드오프 관리 | "3~6년", "미들", 연차 무명시 기본값 |
| **senior** | Proficient~Expert. 표준 수립·후속 양성 | "7년+", "시니어/리드", "아키텍처 책임" |

### 7.2 레벨별 앵커 작성 규칙 (레벨당 5점 = 총 15앵커)

각 레벨의 5점 척도는 §2 알고리즘으로 작성하되, **기대 기준선(3점)을 레벨에 맞게 이동**한다.

| 점수 | junior | mid | senior |
|---:|---|---|---|
| 5 | 미드 수준의 독립 수행·팀 기여 | 시니어 수준의 표준 수립 시도 | 산업·조직 표준 수립, 외부 발표·후속 양성 |
| 4 | 가이드 적게 받고 독립 수행 | 트레이드오프 관리 + 팀 표준 참여 | 조직 표준 주도 + 정량 개선 일관 |
| 3 | 기본 기대를 가이드 하에 수행 | 기본 역할 독립 수행 | 복잡·모호 상황에서 독립 판단 |
| 2 | 핵심 일부 미흡, 잦은 지원 필요 | 일부 영역 미흡, 시니어 의존 | 미드 수준 수행, 시니어 기대 미달 |
| 1 | 기본 개념·기법 부재 | 독립 수행 곤란 | 시니어 역할 기대 크게 미달 |

작성 시 §3 품질 기준(행동 묘사·차등 명확·단일 차원·JD 정합·간결)과 §4 anti-collapse를 레벨마다 동일 적용한다.

### 7.3 JSONB 저장 형식 (`bars_anchors_by_level`)

```json
{
  "junior": {"5":"...","4":"...","3":"...","2":"...","1":"..."},
  "mid":    {"5":"...","4":"...","3":"...","2":"...","1":"..."},
  "senior": {"5":"...","4":"...","3":"...","2":"...","1":"..."}
}
```

`interview_questions`에 기존 `bars_anchors`(단일)와 **공존**한다. 신규 컬럼 `bars_anchors_by_level`(nullable)을 추가하며, 값이 있으면 레벨별을 우선 사용한다.

### 7.4 마이그레이션·폴백 정책 (Critical)

- **하위호환 폴백**: `bars_anchors_by_level`이 NULL인 직무는 기존 단일 `bars_anchors`를 그대로 사용하고, 출력에 "ℹ️ 단일 기준" 주석을 표기한다 (structured-interview-q `output_personalized.md §8`).
- **임의 생성 금지**: 레벨별 앵커가 없을 때 LLM이 즉석에서 레벨별 앵커를 만들어 렌더하지 않는다 (환각 금지). 단일 폴백만 허용.
- **DB 등록 절차**: 레벨별 앵커는 **시니어 패널 검토를 거쳐** 직무별로 점진 등록한다. 우선 5개 직무(frontend/backend/devops/sales/security_engineer) → 검토 → 활성. 실제 스키마 적용·행 분리는 별도 운영 작업이며 `roadmap_v0.2_progress.md`에 기록한다.
- **트랙 명시성**: 렌더되는 레벨은 사용자가 입력한 트랙으로만 결정한다 (LLM의 CV 기반 레벨 추정 금지).
